Создание репозитория

1)Создать репозиторий на удаланнем сервере гитхаб(на сайте)
2)В проекте инициализировать гит:git init(создан локальный гит на компьютере)
3)Добавить все файлы на локальный репоситорий:git add .
4)Зафиксировать все добавленные файлы в локальном репозитории:git commit -m "коментарий"
5)Создать ветку где будут храниться текущая версия проекта:git branch -M {название ветки}
6)подключить удаленный репозиторий к локальному(синхронизировать сайт с папкой в компьютере):git add remote origin {url-адрес}
7)отправить текущее состояние репозитория на удаленный:git push origin {название ветки}


Создание дочерней ветки

1)выполнить переход на ветку с одновременным его созданием:git checkout -b {имя новой ветки} {имя роительской ветки}
2)фиксируем изменение :git commit -am "коментарий"
3)переходим обратно в родительскую ветку:git checkout {имя родительской ветки}
4)встягиваем все изменения из дочерней в родительскую(чтобы родтельская ветка знала о дочерней):git merge --no-ff {имя дочерней ветки}
5)синхронизируем состояние локального репозитория родительской ветки с удаленным(отправляем все изменения на удаленный сервер):git push origin {родительская ветка}
6)синхронизируем состояние локального репозитория дочерней ветки с удаленным(отправляем все изменения на удаленный сервер):git push origin {дочерней ветка}


создание не взаимосвязанных веток
(если проект не подключен к репозиторию то нужно выполнить все первые действия от 1 до 7)
(если проект создается уже инициализированном гите то нужно повторить все действия от 2 до 7 без 6)
1)создаем новую ветку : git branch -M {название новой ветки}
2)синхронизируем состояние локальной ветки с удаленным: git push origin {название новой ветки}


дополнительные команды
1)git branch-показывает на какой ты сейчвс ветке 
2)git checkout {имя ветки}- переход на указанную ветку
3)git reset --hard - возвращает состояние ветки до последнего коммита
4)git merge {название ветки} - переносит все изменения из указанной ветки в текущую
5)git fetch - проверяет актуальность версий ветки с удаленным репозиторием
6)git pull origin {имя текущей или родительской ветки} - переносит все актуальные изменения из удаленной ветки в локальную
